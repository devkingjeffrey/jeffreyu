Open-Source Notion Workflow: Donor & Partner Data Tracker

A single working table for tracking every branch or partner feeding into your reporting — what format they report in, when they last reported, whether their data has been standardized, and whether it's been checked for duplicates against everything else already in the system. This is the same structure referenced in The Reconciliation Tax — a lightweight way to see, at a glance, exactly where the reconciliation gap actually is before it costs you a reporting cycle.

What's in the download

A CSV file, pre-formatted with the columns below and three example rows so you can see it populated before you clear them and add your own branches or partners.

  • Branch / Partner Name — who's reporting
  • Contact Person — who to follow up with directly
  • Region — for filtering and roll-ups
  • Last Report Received — the date, so gaps are visible at a glance
  • Reporting Format — Excel, Google Sheet, PDF, paper log, etc.
  • Currency — flags conversion work needed before consolidation
  • Standardization Status — Standardized / Needs Review / Not Standardized
  • Duplicate Check Done — Yes/No, so nothing gets consolidated twice
  • Notes — anything a teammate would need to know before touching this row

How to set it up in Notion

  1. Download the CSV using the button below.
  2. In Notion, create a new page and choose Import → CSV, then select the file. Notion will build it as a database automatically.
  3. Change the Standardization Status and Duplicate Check Done columns to Select and Checkbox property types respectively, so they're click-to-update instead of free text.
  4. Add a Board view grouped by Standardization Status — this becomes your at-a-glance view of which branches or partners still need cleanup work.
  5. Add a Filtered view for "Last Report Received is more than 30 days ago" to catch branches going quiet before it becomes a reporting-cycle emergency.

Why a tracker like this, not just a bigger spreadsheet

This isn't meant to replace your CRM or become the system of record — it's a lightweight, visible layer that shows the operations team exactly where reconciliation work is needed, before it turns into a scramble against a donor deadline. For the structural fix underneath it — the actual database and automation that keeps this from being a manual tracker in the first place — see CRM Architecture.
